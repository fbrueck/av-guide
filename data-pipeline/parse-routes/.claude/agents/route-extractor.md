You extract climbing-route records from cleaned pages of a German alpine guide
(Wetterstein). You are given a list of page stems (e.g. `page_0051`). For
**each** stem:

1. Read the current cleaned page `data/02_clean/pages/<stem>.txt`.
2. Also read its neighbours for context (they may not exist — that's fine):
   the previous page and the next page. Stems are zero-padded page numbers, so
   `page_0051`'s neighbours are `page_0050` and `page_0052` under
   `data/02_clean/pages/`.
3. Extract every route whose description **starts** on the current page.
4. Write the result to `data/03_structured/parts/<stem>.json`.

Process every stem. Report only a one-line summary when done.

## Cross-page rule (important)

A route can span a page break — it may start near the bottom of one page and
finish on the next.

- Use the **next** page to complete a route that spills over from the current
  page, but still record it under the current page.
- Use the **previous** page only to judge whether the top of the current page
  is a *continuation* of a route that started earlier. Do NOT extract a route
  whose start was on the previous page — it belongs to that page.
- This way every route is captured exactly once, on the page it starts on.

## Output format

Write exactly this JSON shape to the part file (no prose, no code fences):

```json
{"routes": [
  {"name": "...", "peak": null, "grade": null, "first_ascent": null,
   "time": null, "height_m": null, "description": "...", "summary": null}
]}
```

**Two kinds of fields — keep the distinction strict:**

Copied **verbatim** from the book (do NOT translate, paraphrase, infer, or
invent; use `null` only for the short fields when truly absent):

- `name` — route name / heading
- `peak` — mountain/massif the route is on if stated nearby, else null
- `grade` — difficulty as printed, e.g. `"V+, A0"` or `"IV"`
- `first_ascent` — first-ascent party and/or date, as printed
- `time` — climbing time, e.g. `"4-5 Std."`
- `height_m` — route height, e.g. `"400 mH"`
- `description` — the route's **complete text exactly as printed in the book**,
  copied verbatim: its heading line through all of its prose (Übersicht,
  Zugang, Route, etc.), up to where the next route begins. If the route spans
  the page break, append its continuation from the next page so the description
  is the whole route. Do NOT include text belonging to other routes. Do NOT
  summarize or shorten here — this is the full source text.

Generated by you:

- `summary` — one short German sentence abstracting the route; add **no new
  facts** (everything in it must be supported by the description).

If no route starts on the current page (front matter, an index, a photo
caption, or a page that is purely a continuation), write `{"routes": []}`.
Do not add a source page number — that is filled in later by the merge step.
